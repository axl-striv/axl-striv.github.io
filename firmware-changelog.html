STRIV

Firmware Updates

Release notes for Striv Smart Insole firmware. Updates are delivered over-the-air through the Striv app — open the app, go to Settings → Insole, and follow the firmware update prompt.

Most users are on the 1.x production line. If you're on the 2.x early-access SPI hardware (pilot program), see the second section below.

If you have questions, contact axl@striv.run.

1.x — Production line

2026-05-01

v1.2.4 Recommended

Bluetooth identity reliability fix.

  • More reliable Bluetooth identity. The insole now consistently advertises as STRIV_LEFT or STRIV_RIGHT across all Bluetooth scanners and operating systems. Previously, some clients could occasionally see the wrong side label or a generic name due to how the firmware exposed its name to the Bluetooth stack.
  • This is the first Striv firmware shipped with hardware-validated DFU round-trip — the upgrade from v1.2.3 → v1.2.4 (and rollback v1.2.4 → v1.2.3) was verified end-to-end on real hardware before publish.
2026-04-16

v1.2.3

The most polished release in the 1.2 series. Several improvements roll up at once:

  • More reliable post-run sync. When your insole uploads buffered runs after a workout, every step's full data now comes through cleanly — no missing fields if the connection mode changed mid-run.
  • Smarter step detection. The algorithm that decides where each step starts and ends has been rewritten for cleaner, more consistent results across different running styles.
  • Stability fixes across the main loop, sensors, and Bluetooth state machine.
2026-04-07

v1.2.2

Maintenance release. Internal version-string alignment only — no user-visible change versus 1.2.1. If you're on 1.2.1, you can skip ahead to 1.2.3.

2026-04-03

v1.2.1

Bluetooth stability fix.

  • Fixed two latent bugs that could cause occasional disconnects or instability during long sessions: a dangling pointer in the Bluetooth advertising name and a connection reference-count leak.

Recommended over 1.2.0 for everyone.

2026-04-02

v1.2.0

Production-grade hardening.

  • Safer firmware updates. Added a safety timeout to the firmware update flow so a stalled update can't leave the insole in a stuck state.
  • Smoother sensor calibration. The pressure-sensor auto-tuning now adjusts more gradually, reducing noise during the warm-up window.
  • Hardware protection enabled. Production-grade firmware-readout protection is now active.
2026-03-28

v1.1.9

  • Step-detection algorithm tuning for more consistent stride boundaries.

2.x — Early-access SPI hardware (pilot)

The 2.x line targets a newer generation of hardware (SPI-based pressure sensing, different IMU). It's currently distributed to internal testers and pilot users. If your insole is from the standard 1.x line, do not install these — they're for different hardware.

2026-05-01

v2.0.6

Catch-up release: brings the 2.x line up to 1.2.3-equivalent on the SPI hardware, plus the same Bluetooth identity reliability fix as v1.2.4.

  • Same BLE identity fix as v1.2.4 — consistent STRIV_LEFT / STRIV_RIGHT across all scanners.
  • All paired application-layer improvements that landed on the shared codebase between 2.0.5 and now: post-run sync reliability, gait analysis rewrite, miscellaneous bug fixes.
  • Pot-direction fix specific to SPI hardware (was inverted on some boards).
  • Validated end-to-end via hardware DFU round-trip (2.0.5 ↔ 2.0.6 upgrade and rollback both verified).
2026-03-30

v2.0.5

  • Continues 2.0.4 gait detection tuning on SPI hardware.
2026-03-28

v2.0.4

  • Step-detection threshold retune to match the new 416 Hz IMU sampling and shoe-baseline subtraction.
2026-03-27

v2.0.2 / v2.0.3

First 2.x releases — foundational changes for SPI hardware. Install v2.0.3 directly (skip v2.0.2 — same-day hotfix on top).

  • Higher-resolution IMU. Accelerometer sampling raised to 416 Hz (up from 208 Hz); peak-acceleration selection rewritten for physically coherent values.
  • Shoe baseline learning. Per-cell pressure baseline learned automatically during swing phase and subtracted from output — gives cleaner pressure / center-of-pressure data.
  • Touchdown / takeoff detection retuned for SPI hardware response.
  • OTA path enabled end-to-end (mcumgr).

For older firmware versions, see the firmware archive.
